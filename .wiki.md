# Project Summary
This project is a premium, fully responsive personal portfolio website for Sangram Rajpoot, a Full Stack Java Developer. It showcases his skills, projects, and professional journey with smooth animations, an interactive UI, and modern design elements aimed at attracting potential employers.

# Project Module Description
The portfolio website includes the following functional modules:
- **Hero Section**: Features a scroll-based animation displaying a sequence of images showcasing the developer's work.
- **About Me**: Provides a brief introduction and highlights key achievements.
- **Skills**: Displays technical expertise with animated skill cards.
- **Projects**: Showcases featured projects with interactive cards.
- **Experience**: Outlines professional experience in a timeline format.
- **Achievements**: Lists notable accomplishments.
- **Contact**: Includes a functional contact form for inquiries.

# File Description Inventory
- **index.html**: The main structure of the website, containing all sections and linking CSS/JS files.
- **style.css**: Contains styles for the website, including responsive design and animations.
- **script.js**: Handles interactivity, animations, and dynamic content loading, including the fixed image path for the hero section.
- **todo.md**: Outlines design guidelines and tasks for the project.
- **assets/frames/**: Contains PNG images used for the scroll-based animation in the hero section.
- **assets/images/profile.jpeg**: Profile image for the "About Me" section.
- **package.json**: Lists project dependencies and scripts.

# Technology Stack
- **HTML5**: Structure of the website.
- **CSS3**: Styling and layout, including animations.
- **JavaScript**: Interactivity and dynamic content handling.
- **Canvas API**: For rendering the animated hero section.
- **Intersection Observer API**: For scroll animations and lazy loading.

# Usage
1. **Install Dependencies**: Run `npm install` to install required packages.
2. **Build the Project**: Use your build tool of choice (e.g., Webpack, Parcel) to compile the assets.
3. **Run the Project**: Start the development server using your preferred method (e.g., npm start).
